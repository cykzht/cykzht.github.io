1.2.2 Syntax Rules

The switch case statement has the following rules:

  • The type of the result of the expression in the switch statement can be: byte, short, int, or char. Starting from JDK1.5, it supports enum, and starting from Java SE 7, it supports the String type.
  • The switch statement can have multiple case statements. Each case is followed by a value to compare and a colon.
  • The data type of the value in the case statement must be the same as the data type of the expression result, and it can only be a constant or a literal constant.
  • When the value of the expression matches the value of the case statement, the statements after the case statement begin to execute, and the switch statement is exited only when a break statement appears.
  • When a break statement is encountered, the switch statement terminates. The program jumps to the statements after the switch statement to execute. The break statement in the case statement is optional. If there is no break statement, the program will continue to execute the next case statement until a break statement appears or the switch statement completes execution.
  • The switch statement can include a default branch, which is usually the last branch of the switch statement (it can be in any position, but it is recommended to be the last). The default executes when the value of the case statement does not match the value of the expression. The default branch does not require a break statement.
